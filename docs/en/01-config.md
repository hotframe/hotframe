# 1. Configuration: settings, paths, and database (`config/`)

> `config/` is the lowest layer of the framework: it resolves all configuration before any other
> component starts up. If you understand this package, you understand the master controls of
> hotframe.

---

## What this folder is for

`config/` centralizes three orthogonal responsibilities:

1. **Settings** — reading, validating, and exposing application configuration (`settings.py`).
2. **Database** — creating and managing the lifecycle of the async SQLAlchemy engine (`database.py`).
3. **Paths** — resolving the ephemeral filesystem paths used by the framework (`paths.py`).

These three pieces are singletons: they are created once per process and reused throughout the
framework. `bootstrap.py` consumes them in the first steps of startup (lifespan steps 4b and 1,
respectively).

---

## File map

| File | Responsibility |
|---|---|
| [`__init__.py`](../src/hotframe/config/__init__.py) | Docstring only; declares public exports and serves as a readable index of the package. |
| [`settings.py`](../src/hotframe/config/settings.py) | Defines `HotframeSettings` (Pydantic Settings), `get_settings()`, `set_settings()`, and `reset_settings()`. |
| [`database.py`](../src/hotframe/config/database.py) | Provides the `AsyncEngine`, the `async_sessionmaker`, and the FastAPI dependency `get_db()`. |
| [`paths.py`](../src/hotframe/config/paths.py) | Defines `DataPaths`: ephemeral paths under `/tmp/` for media, modules, reports, and cache. |

## `settings.py` — HotframeSettings in depth

### Class `HotframeSettings`

```python
class HotframeSettings(BaseSettings):
    model_config = SettingsConfigDict(
        env_prefix="",
        env_file=".env",
        env_file_encoding="utf-8",
        case_sensitive=False,
        extra="ignore",
    )
```

`HotframeSettings` inherits from `pydantic_settings.BaseSettings`. The user creates a subclass in
their project's `settings.py` and passes the instance to `create_app(settings)`. This allows
adding custom fields without touching the framework.

#### Configuration source precedence

Pydantic Settings resolves values in this order (highest precedence first):

1. Arguments passed to the constructor (`Settings(DATABASE_URL="...")`)
2. Process environment variables
3. `.env` file in the working directory
4. Default values declared in the class

The `extra="ignore"` option means unknown environment variables do not raise an error.
`case_sensitive=False` allows mixing uppercase and lowercase without issues.

### Validators

#### `_normalize_log_level`

```python
@field_validator("LOG_LEVEL")
@classmethod
def _normalize_log_level(cls, v: str) -> str:
    v = v.upper()
    valid = {"DEBUG", "INFO", "WARNING", "ERROR", "CRITICAL"}
    if v not in valid:
        raise ValueError(f"LOG_LEVEL must be one of {valid}, got {v!r}")
    return v
```

Converts the value to uppercase and validates it against the set of valid Python logging levels.

#### `_resolve_path`

```python
@field_validator("MODULES_DIR", "MODULES_CACHE_DIR", mode="before")
@classmethod
def _resolve_path(cls, v: str | Path) -> Path:
    return Path(v).resolve()
```

Converts `MODULES_DIR` and `MODULES_CACHE_DIR` to absolute paths at validation time, removing any
dependency on the current working directory.

#### `_validate_secrets_key`

```python
@model_validator(mode="after")
def _validate_secrets_key(self) -> HotframeSettings:
    if self.DEPLOYMENT_MODE != "local":
        if not self.SECRETS_KEY:
            raise ValueError("SECRETS_KEY is required in non-local deployments. ...")
    if self.SECRETS_KEY:
        decoded = base64.urlsafe_b64decode(self.SECRETS_KEY)
        if len(decoded) != 32:
            raise ValueError(...)
    return self
```

Ensures that any deployment other than `"local"` has a valid `SECRETS_KEY` (exactly 32 decoded
bytes in base64-url format, which is the Fernet key format).

### Computed properties

#### `is_sqlite`

```python
@property
def is_sqlite(self) -> bool:
    return self.DATABASE_URL.startswith("sqlite")
```

Used by `database.py` to skip pool options (SQLite does not support them) and by tests to detect
the CI environment.

#### `is_production`

```python
@property
def is_production(self) -> bool:
    return self.DEPLOYMENT_MODE == "web" and not self.DEBUG
```

Determines whether the log format is forced to JSON and whether Swagger documentation is disabled.

### Module functions

#### `get_settings() -> HotframeSettings`

Lazy singleton. The first call instantiates `HotframeSettings()` by reading the environment;
subsequent calls return the same instance. This is the function used internally throughout the
framework.

#### `set_settings(settings: HotframeSettings) -> None`

Called by `create_app(settings)` to inject the instance built by the project. This lets the
project override the singleton before any part of the framework reads it.

#### `reset_settings() -> None`

Sets the singleton back to `None`. For tests only: forces the next call to `get_settings()` to
rebuild the instance from the current environment.

## `database.py` — Async engine and sessions

### Module-level variables

```python
_engine: AsyncEngine | None = None
_session_factory: async_sessionmaker[AsyncSession] | None = None
```

Module-level singletons. Initialized on the first call and destroyed by `dispose_engine()`.

---

### `get_engine() -> AsyncEngine`

```python
def get_engine() -> AsyncEngine:
    global _engine
    if _engine is None:
        settings = get_settings()
        kwargs: dict = {"echo": settings.DB_ECHO}
        if not settings.is_sqlite:
            kwargs.update(
                pool_size=settings.DB_POOL_SIZE,
                max_overflow=settings.DB_MAX_OVERFLOW,
                pool_recycle=settings.DB_POOL_RECYCLE,
                pool_pre_ping=True,
                pool_timeout=settings.DB_POOL_TIMEOUT,
            )
            if settings.DB_DISABLE_PREPARED_STATEMENTS and "asyncpg" in settings.DATABASE_URL:
                kwargs["connect_args"] = {
                    "prepared_statement_cache_size": 0,
                    "statement_cache_size": 0,
                }
        else:
            kwargs["connect_args"] = {"check_same_thread": False}
        _engine = create_async_engine(settings.DATABASE_URL, **kwargs)
    return _engine
```

Key points:
- `pool_pre_ping=True` — before handing out a connection from the pool, executes a `SELECT 1` to
  verify it is still alive. Prevents silent errors after a network disconnection.
- When `DB_DISABLE_PREPARED_STATEMENTS=True` and the driver is asyncpg, passes
  `prepared_statement_cache_size=0` and `statement_cache_size=0` in `connect_args`. This is
  required with transaction-mode poolers (RDS Proxy, PgBouncer, Supavisor) that rotate the
  connection between transactions, invalidating the client's prepared-statement cache.
- SQLite receives `check_same_thread=False` and no pool options.

---

### `get_session_factory() -> async_sessionmaker[AsyncSession]`

```python
def get_session_factory() -> async_sessionmaker[AsyncSession]:
    global _session_factory
    if _session_factory is None:
        _session_factory = async_sessionmaker(
            bind=get_engine(),
            class_=AsyncSession,
            expire_on_commit=False,
        )
    return _session_factory
```

`expire_on_commit=False` is deliberate: ORM objects are not marked as expired after a commit,
meaning you can continue accessing their attributes without triggering another query. This is
essential in async handlers where the session is already closed by the time the response is
serialized.

---

### `get_db() -> AsyncGenerator[AsyncSession, None]`

```python
async def get_db() -> AsyncGenerator[AsyncSession, None]:
    factory = get_session_factory()
    async with factory() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
```

FastAPI dependency. Each request receives a fresh session, which is committed automatically on
success and rolled back on exception. Typical usage:

```python
from hotframe.config.database import get_db
from fastapi import Depends
from sqlalchemy.ext.asyncio import AsyncSession

@router.get("/items")
async def list_items(db: AsyncSession = Depends(get_db)):
    ...
```

---

### `dispose_engine() -> None`

```python
async def dispose_engine() -> None:
    global _engine, _session_factory
    if _engine is not None:
        await _engine.dispose()
        _engine = None
        _session_factory = None
```

Called by `bootstrap.py` during lifespan shutdown. Closes all pool connections gracefully and
clears the singletons, which allows the process to be reused cleanly in tests.

## `paths.py` — Ephemeral system paths

### Design philosophy

hotframe assumes a container environment: the filesystem is **100% ephemeral**. Persistent data
goes to S3 or the database. All local paths are caches that are rebuilt on the next startup.

### Class `DataPaths`

```python
class DataPaths:
    def __init__(self, base: Path | None = None) -> None:
        if base is not None:
            self._base = base.resolve()
        elif env := os.environ.get("DATA_PATH"):
            self._base = Path(env).resolve()
        else:
            self._base = Path("/tmp/hotframe-data")
```

The base directory is resolved in this order: explicit argument → `DATA_PATH` environment variable
→ `/tmp/hotframe-data`.

All attributes are `@cached_property`, meaning the path is constructed only once per instance:

| Property | Path | Use |
|---|---|---|
| `base` | `_base` (configurable) | Root directory for ephemeral data. |
| `media` | `/tmp/hotframe-media` | Media cache in dev. In production, media goes to S3. |
| `modules` | `/tmp/modules` | Cache for module code downloaded from S3. Rebuilt on each cold start. |
| `reports` | `_base/reports` | Temporary directory for report generation. Finished reports go to S3. |
| `temp` | `_base/temp` | General-purpose temporary directory. |
| `cache` | `_base/cache` | General-purpose cache. |

### `ensure_dirs() -> None`

```python
def ensure_dirs(self) -> None:
    for d in self.all_dirs:
        d.mkdir(parents=True, exist_ok=True)
```

Creates all paths if they do not exist. Called from initialization code when something needs to be
written to disk before the directory has been created.

### `all_dirs: list[Path]`

A (non-cached) property that returns the list of all relevant paths. Used by `ensure_dirs()` and
by tests to verify that the setup is correct.

### Module functions

#### `get_data_paths() -> DataPaths`

Lazy singleton. Analogous to `get_settings()` but for paths. Always returns the same instance
within the process.

#### `reset_data_paths() -> None`

For tests only. Sets the singleton back to `None`.

## How this fits with the rest of the framework

`config/` is the lowest layer and does not import any other hotframe subpackage. The rest of the
framework imports it:

- **`bootstrap.py`** calls `get_settings()` in lifespan step 4b, `get_engine()` in step 1, and
  `dispose_engine()` on shutdown.
- **`discovery/scanner.py`** does not import `config/` directly, but the orchestrator that calls
  it (the engine) reads `settings.MODULES_DIR` to know where to scan.
- **`apps/registry.py`** and **`engine/module_runtime.py`** read `get_settings()` to configure
  the runtime.
- **All repositories and services** receive the session via `get_db()` as a FastAPI dependency.

## Gotchas and design decisions

**`SECRET_KEY` is regenerated on every startup if not set.** The default is
`secrets.token_urlsafe(64)`, which produces a different value in every process. In production you
**must** set `SECRET_KEY` explicitly, or signed sessions will be invalidated after a restart.

**`DB_DISABLE_PREPARED_STATEMENTS` is not optional with transaction-mode poolers.** If you use
RDS Proxy, PgBouncer in transaction mode, or any proxy that rotates the connection between
transactions, asyncpg caches prepared statements per TCP connection and fails when it receives a
recycled connection. Enable this flag.

**`expire_on_commit=False` is intentional.** In async frameworks, accessing ORM object attributes
after the session has closed would raise `MissingGreenlet` or `DetachedInstanceError`. This flag
prevents SQLAlchemy from marking attributes as expired on commit.

**Paths are ephemeral by design.** Do not store anything important under `/tmp/hotframe-*`. If the
container restarts, that data is gone. Code that uses `DataPaths` assumes that those paths may not
exist at any given moment — that is exactly why `ensure_dirs()` exists and is called before
writing.

**`INSTALLED_APPS` does not exist in hotframe.** Apps are discovered automatically from `apps/`.
You can use `EXTRA_ROUTERS` to mount additional routers that do not follow the conventional folder
structure.
