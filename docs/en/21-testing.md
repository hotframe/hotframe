# 21. Testing utilities (testing/)

> `testing/` provides the pieces you need to write tests for your hotframe application without spinning up a real database, without external infrastructure, and without the full middleware stack that only makes sense in production. The goal is for a test's setup to be three lines and for assertions to be straightforward.

---

## What this folder is for

Testing a FastAPI + SQLAlchemy application involves well-known friction points: creating the async engine for tests, ensuring tables exist before a test and are cleaned up afterwards, disabling CSRF and rate limiting so they do not complicate test requests, and having doubles (fakes) for components that produce side effects (event bus, hook registry). `testing/` solves exactly that friction with a small, orthogonal set of utilities.

---

## File map

| File | Responsibility |
|---|---|
| [`testing/__init__.py`](../src/hotframe/testing/__init__.py) | All subpackage logic. Exports `create_test_app`, `test_db_session`, `create_test_tables`, `drop_test_tables`, `cleanup_test_db`, `FakeEventBus`, `FakeHookRegistry`. |
| [`testing/test_lazy_imports.py`](../src/hotframe/testing/test_lazy_imports.py) | Regression test for the hotframe package: verifies that every name in `hotframe.__all__` resolves without an import error. |

## Global test engine state

```python
_test_engine = None
_test_session_factory = None
```

Two module-level variables that act as singletons for the engine and factory across the entire test session. This is intentional: creating a new in-memory SQLite engine for every test would be expensive and could cause file contention. The recommended pattern is:

- Create the engine once per pytest session (fixture with `scope="session"`).
- Use rollback at the end of each test to isolate state.

## `create_test_app(settings, **overrides)`

**Signature:**
```python
def create_test_app(settings: Any | None = None, **overrides: Any) -> FastAPI
```

Creates a FastAPI application configured for testing.

### Test defaults

If no `settings` object is passed, the following values are applied automatically:

| Setting | Test value | Reason |
|---|---|---|
| `DATABASE_URL` | `"sqlite+aiosqlite:///:memory:"` | No file, no external server |
| `DEBUG` | `True` | Full error messages |
| `DEPLOYMENT_MODE` | `"local"` | Disables production-only behaviors |
| `SECRET_KEY` | `"test-secret-key-not-for-production"` | Fixed value for signed cookies |
| `CSRF_EXEMPT_PREFIXES` | `["/"]` | Exempts **all** routes from CSRF |
| `RATE_LIMIT_API` | `999999` | Eliminates rate limiting |
| `RATE_LIMIT_AUTH` | `999999` | Eliminates auth rate limiting |
| `LOG_LEVEL` | `"WARNING"` | Suppresses INFO/DEBUG logs in tests |

The `**overrides` are merged on top of the defaults before the `HotframeSettings` object is constructed:

```python
test_defaults.update(overrides)
settings = HotframeSettings(**test_defaults)
```

### Internal flow

```python
set_settings(settings)          # registers globally so create_app can use it
app = create_app(settings)      # full bootstrap
return app
```

The function calls the same `create_app` used by the real application, which ensures that tests exercise the same stack (middlewares, Jinja2, ComponentRegistry, etc.).

### Usage example

```python
# tests/conftest.py
import pytest
from hotframe.testing import create_test_app

@pytest.fixture
async def app():
    return create_test_app()

# Specific override
@pytest.fixture
async def app_with_cors():
    return create_test_app(CORS_ORIGINS=["http://localhost:3000"])
```

## `test_db_session()`

**Signature:**
```python
async def test_db_session() -> AsyncGenerator[AsyncSession, None]
```

Async generator that yields a SQLAlchemy session connected to an in-memory SQLite database. Creates tables on first use and rolls back at the end of each session.

### Lifecycle

```python
# First call: creates the engine and tables
if _test_engine is None:
    _test_engine = create_async_engine("sqlite+aiosqlite:///:memory:", echo=False)
    async with _test_engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)

# Every call: yields a session and rolls back
async with _test_session_factory() as session:
    try:
        yield session
    finally:
        await session.rollback()
```

The rollback in `finally` ensures every test starts with a clean database even if the previous test failed midway.

### Usage example

```python
# tests/conftest.py generated by hf startproject
@pytest.fixture
async def db(app):
    async for session in test_db_session():
        yield session

# Real test
async def test_create_user(db):
    user = User(email="test@example.com")
    db.add(user)
    await db.flush()
    assert user.id is not None
```

**Note:** the `db` fixture depends on `app` because `create_test_app()` is what calls `set_settings()`, ensuring the engine is configured before `test_db_session()` tries to use it.

## `create_test_tables()`

**Signature:** `async def create_test_tables() -> None`

Explicitly creates all tables in `Base.metadata` on the test engine. Useful when a test does not go through `test_db_session()` but still needs the tables to exist (e.g., migration tests or data fixture tests).

```python
async def create_test_tables() -> None:
    global _test_engine
    if _test_engine is None:
        _test_engine = create_async_engine("sqlite+aiosqlite:///:memory:", echo=False)
    async with _test_engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
```

---

## `drop_test_tables()`

**Signature:** `async def drop_test_tables() -> None`

Drops all tables from the test engine. Useful for session-fixture teardown or for resetting the schema between test groups.

---

## `cleanup_test_db()`

**Signature:** `async def cleanup_test_db() -> None`

Disposes of the engine and resets the `_test_engine` and `_test_session_factory` singletons to `None`. Intended for session-scoped fixture teardown (`scope="session"`):

```python
@pytest.fixture(scope="session", autouse=True)
async def db_cleanup():
    yield
    await cleanup_test_db()
```

Calls `engine.dispose()` to close all pool connections before resetting state. After `cleanup_test_db()`, the next call to `test_db_session()` will create a fresh engine (and fresh tables).

## `FakeEventBus`

**Signature:**
```python
class FakeEventBus:
    events: list[tuple[str, Any]]
    typed_events: list[Any]

    async def emit(self, event_name: str, data: Any = None, **kwargs: Any) -> None
    async def emit_typed(self, event: Any) -> None
    def reset(self) -> None
```

An in-memory event bus for tests. Instead of running real subscribers, it **only records** the events emitted. Enables assertions like:

```python
bus = FakeEventBus()

# Inject into the app / service under test
await service.create_invoice(bus=bus, ...)

# Assert what was emitted
assert bus.events == [("invoice.created", {"invoice_id": 42})]
assert len(bus.typed_events) == 1
assert isinstance(bus.typed_events[0], InvoiceCreated)
```

### `emit(event_name, data, **kwargs)`

Records a `(event_name, data)` tuple in `self.events`. Ignores `**kwargs` (which the real bus uses for named payloads). Note: if the code under test calls `bus.emit("invoice.paid", invoice_id=42)` — i.e., with no `data` but with kwargs — `data` will be `None` in the recorded tuple. The kwargs are lost in the fake.

### `emit_typed(event)`

Records the typed event object in `self.typed_events`. Enables assertions on the type and attributes of the event.

### `reset()`

Clears both lists. Useful for reusing the same instance across multiple tests or assertions:

```python
bus = FakeEventBus()

await op1(bus=bus)
assert ("event.1", ...) in bus.events
bus.reset()

await op2(bus=bus)
assert ("event.2", ...) in bus.events
```

## `FakeHookRegistry`

**Signature:**
```python
class FakeHookRegistry:
    _actions: dict[str, list]
    _filters: dict[str, list]

    async def do_action(self, name: str, *args: Any, **kwargs: Any) -> None
    async def apply_filters(self, name: str, value: Any, *args: Any, **kwargs: Any) -> Any
    def add_action(self, name: str, fn: Any, priority: int = 10) -> None
    def add_filter(self, name: str, fn: Any, priority: int = 10) -> None
```

An in-memory hook registry that **does execute** registered functions (unlike `FakeEventBus`, which only records). This lets you test code that depends on hooks using controlled test handlers.

### `add_action(name, fn, priority=10)`

Registers `fn` for hook `name`. `priority` is accepted for signature compatibility but is not used — handlers are executed in registration order.

### `do_action(name, *args, **kwargs)`

Invokes all handlers registered for `name`:

```python
async def do_action(self, name: str, *args: Any, **kwargs: Any) -> None:
    for fn in self._actions.get(name, []):
        await fn(*args, **kwargs)
```

If no handlers are registered for that name, it does nothing (no error is raised).

### `add_filter(name, fn, priority=10)`

Registers `fn` as a filter for `name`.

### `apply_filters(name, value, *args, **kwargs)`

Passes `value` through all registered filters in registration order:

```python
async def apply_filters(self, name: str, value: Any, ...) -> Any:
    for fn in self._filters.get(name, []):
        value = await fn(value, *args, **kwargs)
    return value
```

### Usage example

```python
hooks = FakeHookRegistry()

# Register a test filter that doubles the subtotal
hooks.add_filter("invoice.subtotal", lambda total, ctx: total * 2)

# Assert that the system applies the filter
result = await hooks.apply_filters("invoice.subtotal", 100.0, ctx={})
assert result == 200.0
```

## `test_lazy_imports.py` — package regression test

**File:** [`testing/test_lazy_imports.py`](../src/hotframe/testing/test_lazy_imports.py)

```python
def test_all_lazy_imports_resolve():
    import hotframe

    for name in hotframe.__all__:
        assert getattr(hotframe, name) is not None  # force the lazy import to resolve
```

This test has a very specific purpose: to verify that **no name declared in `hotframe.__all__`** raises an `ImportError` or `AttributeError`, or returns `None` when resolved. The `hotframe` package uses lazy imports (names in the public namespace are defined with descriptors or `__getattr__` to avoid loading heavy modules on `import hotframe`). This test acts as a safety net: if someone adds a name to `__all__` but forgets to implement the resolution, the test fails.

**When it can fail:**
- When a new export is added to `hotframe.__all__` without implementing the corresponding `__getattr__`.
- When an internal module is renamed without updating the lazy import system.
- When a circular dependency prevents a module from loading.

**Why it lives in `testing/`:**
This test is part of the hotframe package itself (not a user application). Its presence in `testing/` indicates that the framework's own internal tests also live here, alongside the utilities for user applications.

## How it fits into the rest of the framework

| Component | Relationship with `testing/` |
|---|---|
| `hotframe.bootstrap.create_app` | `create_test_app` calls it directly — tests exercise the real bootstrap |
| `hotframe.config.settings.HotframeSettings` / `set_settings` | `create_test_app` calls `set_settings` so the rest of the framework sees the test settings |
| `hotframe.models.base.Base` | `create_test_tables` and `test_db_session` use `Base.metadata` to create/drop tables |
| `hotframe.signals.dispatcher.AsyncEventBus` | `FakeEventBus` has the same public interface (`emit`, `emit_typed`); it is injected wherever real code expects `AsyncEventBus` |
| `hotframe.signals.hooks.HookRegistry` | `FakeHookRegistry` replicates the `HookRegistry` interface |
| `management/cli.py` → `startproject` | The `conftest.py` generated by `hf startproject` already uses `create_test_app` and `test_db_session` — the scaffold and the utilities are kept in sync |

## Recommended `conftest.py` pattern

The `conftest.py` generated by `hf startproject` is the starting point:

```python
# tests/conftest.py
import pytest
from hotframe.testing import create_test_app, test_db_session

@pytest.fixture
async def app():
    return create_test_app()

@pytest.fixture
async def db(app):
    async for session in test_db_session():
        yield session
```

For a more advanced project, add session-scoped fixtures and cleanup:

```python
import pytest
from hotframe.testing import (
    create_test_app, test_db_session, cleanup_test_db,
    FakeEventBus, FakeHookRegistry,
)

@pytest.fixture(scope="session")
async def app():
    return create_test_app()

@pytest.fixture(scope="session", autouse=True)
async def _cleanup(app):
    yield
    await cleanup_test_db()

@pytest.fixture
async def db(app):
    async for session in test_db_session():
        yield session

@pytest.fixture
def event_bus():
    return FakeEventBus()

@pytest.fixture
def hooks():
    return FakeHookRegistry()
```

## Gotchas and design decisions

**1. `CSRF_EXEMPT_PREFIXES = ["/"]` exempts all routes.**
In production this setting only exempts `/api/`, `/health`, and `/static/`. In tests everything is exempted so you do not have to include a CSRF token in every test request. If you specifically want to test CSRF protection, pass `CSRF_EXEMPT_PREFIXES=[]` as an override to `create_test_app`.

**2. In-memory SQLite does not persist between tests.**
`test_db_session()` rolls back after each test, but the in-memory SQLite engine does persist tables across tests in the same pytest session. This is the intended behavior: tables are created once and each test operates inside a transaction that is discarded.

**3. `FakeEventBus` does not execute subscribers.**
Unlike `FakeHookRegistry`, `FakeEventBus` does not call registered handlers — it only accumulates events. If the code under test **relies** on subscribers running (e.g., an email sent as a result of an event), you need to register those subscribers on the fake manually, or use the real bus in an integration fixture.

**4. `FakeHookRegistry` ignores `priority`.**
The real `HookRegistry` sorts handlers by priority. The fake executes them in registration order. If you have logic that depends on priority ordering, the fake may produce false positives in tests.

**5. The `db` fixture must depend on `app`.**
`test_db_session()` requires `set_settings()` to have been called first (to know which database URL to use). `create_test_app()` calls `set_settings()`. Therefore the `db` fixture must declare `app` as a dependency, even if it does not use it directly. If `db` is called without `app`, the test engine may be created with hotframe's default settings rather than the project's.

**6. `test_lazy_imports.py` as a contract test.**
Located in `testing/` but it tests hotframe itself, not user applications. It acts as a contract test for the public namespace: every name in `__all__` must be resolvable. Its placement in `testing/` (rather than a top-level `tests/` directory) is a packaging decision — it is part of the installable package and can be run in the user's environment to verify installation integrity.
