# Security policy

## Supported versions

`hotframe` follows semantic versioning. The current major's latest
minor receives security patches; previous minors receive patches for
six months after a new minor ships.

## Reporting a vulnerability

**Do not open a public issue.** Use one of the following private
channels:

- GitHub: open a [private security advisory](https://github.com/hotframe/hotframe/security/advisories/new).
- Email: `security@hotframe.dev`.

Please include:

1. A description of the vulnerability.
2. Steps to reproduce.
3. The potential impact.
4. Your contact details (optional, only if you want credit).

## Response SLA

- Acknowledgement: within 48 business hours.
- Initial assessment: within 7 days.
- Patch for critical vulnerabilities: within 14 days of confirmation.
- Coordinated disclosure: public after the patch ships, with a 30-day
  grace period (or shorter if agreed).

## Credit

Valid reports are credited in [CHANGELOG.md](CHANGELOG.md) and on
the corresponding GitHub Security Advisory, with the reporter's
permission.
